# GitHub Proof Checklist

Generated on 2026-07-05.

Use this file when someone asks for proof that the project was uploaded and is technically traceable.

## What To Show

- Public/private GitHub repository URL.
- Latest commit hash.
- Commit date and branch.
- `README.md` project overview.
- `docs/06_project_worklog.md` for chronological work evidence.
- `docs/13_training_run_log.md` for YOLO training evidence and honest results.
- `docs/14_object_avoidance_route_handling.md` for object avoidance and route handling.
- `scripts/run_all_verification.ps1` for repeatable verification.

## What Is In Git

- Core Python source under `src/facade_uav/`.
- ROS 2 package scaffold under `ros2_ws/src/`.
- Gazebo/SDF simulation files.
- Dataset conversion, training, demo, and verification scripts.
- Architecture, runbook, status, training, and evaluation documentation.

## What Is Intentionally Not In Git

Large/generated artifacts are ignored:

- Downloaded datasets under `data/`.
- Trained model weights under `models/`.
- Generated images/reports under `outputs/`.
- Ultralytics run folders under `runs/`.
- ROS build/install/log folders.
- Downloaded YOLO base weights.

These can be regenerated by following the scripts and docs. This keeps GitHub clean and avoids uploading bulky generated files.

## Verification Command

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_all_verification.ps1
```
